# Staleks ERP — Пошаговый walkthrough: Заказы

## Подготовка

1. Убедитесь что Docker запущен: `docker compose ps`
2. Все 4 контейнера должны быть **Up**: `staleks_backend`, `staleks_frontend`, `staleks_postgres`, `staleks_redis`
3. Если нет — `docker compose up -d && make migrate && make seed`

**URL:** http://localhost:3000
**Логин:** `owner` / `ChangeMe123!`

---

## Шаг 1. Вход в систему

1. Откройте **http://localhost:3000**
2. Введите логин: `owner`, пароль: `ChangeMe123!`
3. Нажмите **"Войти"**
4. Вы попадёте на Dashboard

## Шаг 5. Прикрепление конфигураций

### Сначала создайте конфигурацию (если ещё нет):
1. Откройте **"Конфигуратор"** в боковом меню
2. Нажмите **"+ Новая конфигурация"**
3. Выберите тип двери: **Техническая** или **С Отделкой**
4. Заполните параметры (модель, размеры, цвет, замок и т.д.)
5. Сохраните

### Прикрепите к заказу:
1. Вернитесь в заказ (Заказы → выберите ваш заказ)
2. В секции "Конфигурации дверей" внизу появится **"Прикрепить конфигурацию из конфигуратора"**
3. Нажмите на нужную конфигурацию — она прикрепится
4. Финансовая сводка автоматически пересчитается

### Кнопки на каждой конфигурации:
- **📋 (Copy)** — Дублировать конфигурацию (создаёт копию для редактирования)
- **✕ (X)** — Открепить конфигурацию от заказа

## Шаг 6. Workflow — продвижение по статусам

### Жизненный цикл заказа:
```
Черновик → Замер → Подтверждён → В производстве → Отгружен → Завершён
                                                              ↘ Отменён (с любого этапа)
```

### Переход по статусам:

| Текущий статус | Кнопка | Что происходит |
|---|---|---|
| **Черновик** | "Отправить на замер" | Статус → Замер |
| **Замер** | "Подтвердить заказ" | Статус → Подтверждён. **Цены фиксируются!** |
| **Подтверждён** | "Запустить в производство" | Статус → В производстве. Конфигурации каскадно → in_production |
| **В производстве** | "Отметить отгруженным" | Статус → Отгружен |
| **Отгружен** | "Завершить заказ" | Статус → Завершён |

### Что происходит при каждом переходе:
- **confirmed_at** заполняется автоматически при подтверждении
- **production_started_at** — при запуске в производство
- **shipped_at** — при отгрузке
- **completed_at** — при завершении

### Фиксация цен (Блок 7):
Когда заказ переходит в статус **"Подтверждён"**:
- На каждой конфигурации `locked_price = price_estimate`
- После этого даже если price_estimate изменится — в заказе останется старая цена
- В UI рядом с ценой появляется бейдж **(зафикс.)**

---

## Шаг 7. Редактирование доп. услуг

В статусах Черновик / Замер / Подтверждён можно редактировать:
1. **Стоимость замера** — введите сумму, при потере фокуса (Tab или клик вне поля) сохраняется автоматически
2. **Доставка** — аналогично
3. **Монтаж** — аналогично

Финансовая сводка пересчитывается автоматически после каждого изменения.

---

## Шаг 8. Финансовая сводка с НДС

Формула расчёта:
```
subtotal = SUM(locked_price ?? price_estimate) * quantity)  для каждой конфигурации
discount = subtotal * discount_percent / 100
total_before_vat = subtotal - discount + замер + доставка + монтаж
vat = total_before_vat * vat_rate / 100
TOTAL = total_before_vat + vat
outstanding = TOTAL - prepayment
```

Все суммы отображаются в **тенге (₸)** — валюта Казахстана.

## Новые возможности (Sprint 1)

### Что добавлено по сравнению с предыдущей версией:

1. **Статус "Замер"** — новый этап между Черновик и Подтверждён
2. **Канал продаж** — corporate/dealer/retail
3. **Объект** — для B2B заказов (ЖК "Нео Парк")
4. **Замерщик** — выбор из списка пользователей с ролью "measurer"
5. **Дополнительные услуги** — замер, доставка, монтаж (с суммами)
6. **НДС 16%** — автоматический расчёт в финансовой сводке
7. **Фиксация цен** — при подтверждении заказа
8. **Источник заявки** — откуда пришёл клиент
9. **Таймлайн дат** — автоматическое заполнение confirmed_at, shipped_at и т.д.
10. **Валюта ₸ (тенге)** — вместо ₽ (рубль)
11. **Дублирование конфигураций** — кнопка 📋 на каждой конфигурации
12. **Workflow через API** — `PATCH /orders/{id}/status` вместо прямого update
